# 🤖 Robotic Arena Tracker (Система комп'ютерного зору)

Високопродуктивна серверна система комп'ютерного зору на **Rust** для відстеження роботизованої платформи та перешкод на випробувальному стенді. Система обробляє відеопотік, обчислює координати об'єктів та відправляє телеметрію на робота по протоколу UDP у реальному часі.

## ✨ Архітектура та технічні рішення

Проект вирішує низку класичних проблем комп'ютерного зору та роботи з обладнанням у нестабільних умовах.

### 1. Захоплення відеопотоку (V4L2)
Для забезпечення мінімальної затримки використовується пряме читання з камери через біндинги OpenCV до підсистеми Video for Linux 2 (V4L2). У коді реалізовано автоматичний сканер портів `/dev/video*`, який обходить блокування монопольного доступу ОС та підключається до активного пристрою.


### 2. Гібридний трекінг та Математична морфологія
Дешеві матриці веб-камер та глянцевий пластик робота створюють сильні оптичні перешкоди (сині відблиски від вікна та "брудні" тіні на зеленому тлі). 
Для стабільного трекінгу алгоритм конвертує кадр у простір **HSV** та використовує пошук за темним силуетом. Щоб "склеїти" глянцевий корпус та білі дроти в єдиний об'єкт, застосовується математична морфологія:
* **Дилатація (Dilation x6):** Сильне розширення темних пікселів для поглинання дротів.
* **Ерозія (Erosion x2):** Звуження контурів для відсікання дрібного піксельного шуму матриці.


### 3. Динамічний радар перешкод
Логіка уникнення перешкод винесена на сервер, щоб розвантажити мікроконтролер робота. Алгоритм сортує всі об'єкти на стенді за площею контуру.
Для кожної знайденої перешкоди обчислюється Евклідова відстань до центру мас робота:
`d = √((x2 - x1)² + (y2 - y1)²)`
Об'єкти, що знаходяться в радіусі $\le 20.0$ см, класифікуються як критичні (червона рамка) і передаються в пакет телеметрії.


### 4. Інтерактивне калібрування (Live Calibration Tool)
Через різницю в апаратних профілях (ISP) різних камер, жорстко задані фільтри HSV часто дають збій при зміні освітлення. У систему вбудовано модуль перехоплення подій клавіатури для налаштування фільтрів "на льоту" без перекомпіляції коду:
* `[ W ]` / `[ S ]` — Коригування верхнього порогу яскравості (V Max)
* `[ E ]` / `[ D ]` — Коригування верхнього порогу кольору (S Max)


### 5. UDP-Телеметрія
Передача координат здійснюється за протоколом UDP для забезпечення мінімальної затримки (Low Latency) в системах реального часу. 
Формат формованого пакету (30 разів на секунду):
`RB:<x>,<y>;OB:<x1>,<y1>|<x2>,<y2>\n`
*(Приклад: `RB:55.2,30.5;OB:60.1,25.0|50.0,35.2`)*

## 🛠 Встановлення та запуск (Ubuntu / Linux)

### 1. Встановлення системних залежностей
```bash
sudo apt update
sudo apt install libopencv-dev clang libclang-dev libv4l-dev v4l-utils


2. Збірка проекту
Bash
git clone <URL_ВАШОГО_РЕПОЗИТОРІЮ>
cd <ІМ'Я_ПАПКИ>
cargo build --release

3. Налаштування мережі
Вкажіть актуальну IP-адресу вашого робота у файлі src/main.rs:
Rust
let robot_ip = "192.168.X.X:8888";

4. Запуск
Bash
cargo run --release
